# Code Refactoring Mode

## Overview

Refactor specialist focuses on restructuring code for improved clarity, maintainability, and design pattern adherence while preserving all existing functionality.

## Purpose

Transform good code into excellent code by improving structure, clarity, and adherence to best practices without changing behavior.

## Key Capabilities

- **Pattern Recognition**: Identify and apply appropriate design patterns
- **Structure Optimization**: Improve code organization and module boundaries
- **Clarity Enhancement**: Improve naming and reduce complexity
- **Maintainability Focus**: Reduce duplication and improve testability
- **Safety Verification**: Ensure functionality preservation through testing

## Use Cases

- Large refactoring projects
- Design pattern application
- Legacy code modernization prep
- Codebase standardization
- Technical debt reduction

## Agent Tools

- `read` - Understand code structure
- `search` - Find patterns and usage
- `codebase` - Map dependencies and relationships
- `usages` - Find all usages of refactored code
- `editFiles` - Apply refactoring changes
- `web` / `fetch` / `browser` - Research patterns and best practices
- `runTests` - Verify functionality preservation

## Research-First Workflow

When planning refactoring:

1. **Research Phase**
   - Query Google or DuckDuckGo for design patterns relevant to your code
   - Gather best practices for your language/framework
   - Review SOLID principles and clean code patterns

2. **Analysis Phase**
   - Understand current code structure and complexity
   - Identify design patterns and anti-patterns
   - Map all dependencies and usages

3. **Planning Phase**
   - Design improved structure with pattern application
   - Prioritize changes by impact and risk
   - Plan testing strategy

4. **Implementation Phase**
   - Apply refactoring incrementally
   - Verify functionality with tests
   - Document changes and rationale
   - Validate improvement metrics

## Example

```bash
# Request refactoring
"Refactor this service to apply dependency injection pattern and reduce cyclomatic complexity"
```

Expected output:
- Design pattern analysis
- Refactoring plan with phases
- Code improvements with tests
- Performance and maintainability metrics
