## Purpose

Product documentation agents are designed to review and improve technical documentation for specific software systems, with focus on:

- Implementation traceability
- Developer clarity
- Stakeholder understanding
- Code-synchronized accuracy
- Architecture honesty

## How to Use

Place new agent files in this directory following the naming convention: `[product-name]-documenter.agent.md`

Each agent should include:
- Clear description of target audience (developers, stakeholders, etc.)
- Specific documentation rules and requirements
- Requirements for implementation traceability
- Output format specifications
