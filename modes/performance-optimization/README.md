# Performance Optimization Mode

## Overview

The Performance Optimization mode analyzes code and systems for efficiency, scalability, and bottleneck identification with concrete optimization recommendations.

## Files

- **`performance-optimizer.agent.md`** - Performance optimization specialist

## Target Audience

- Performance engineers
- Database administrators  
- DevOps and infrastructure specialists
- Scalability architects
- Backend engineers

## Optimization Dimensions

- **Algorithm Complexity**: Big O analysis, inefficient patterns
- **Database**: Query optimization, indexes, N+1 problems
- **Memory & Resources**: Leaks, GC pressure, allocation efficiency
- **Concurrency**: Parallelization, thread contention, async patterns
- **Caching**: Hit rates, staleness, invalidation strategy
- **Scalability**: Horizontal/vertical scaling, sharding, load distribution

## Benchmarking & Metrics

For each optimization:
- Current performance baseline
- Optimized performance target
- Percentage improvement
- Implementation effort (hours)
- ROI calculation (improvement per effort)

## Scalability Roadmap

Provides path to 10x growth with:
- Current throughput analysis
- Scaling options ranked by ROI
- Phased implementation approach
- Monitoring and regression detection

## Use Cases

- Performance profiling and analysis
- Database query optimization
- Scalability planning
- Bottleneck investigation

## Research-First Workflow

When optimizing performance:

1. **Research Phase**
   - Query Google or DuckDuckGo for optimization patterns in your language/framework
   - Gather benchmarking methodologies and tools
   - Review case studies of similar systems at scale

2. **Profiling Phase**
   - Run performance tests with current baseline
   - Identify bottlenecks and resource constraints
   - Analyze algorithm complexity and data flow

3. **Optimization Phase**
   - Research alternative approaches for identified bottlenecks
   - Calculate ROI for each optimization
   - Implement changes incrementally

4. **Validation Phase**
   - Measure performance improvements
   - Verify no regression in other areas
   - Document optimization decisions
- Microservices optimization
- Cost reduction through efficiency
