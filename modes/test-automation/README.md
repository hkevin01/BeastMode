# Test Automation Mode

## Overview

The Test Automation mode designs comprehensive testing strategies with coverage planning, automation architecture, and CI/CD integration.

## Files

- **`test-specialist.agent.md`** - Test automation specialist

## Target Audience

- QA engineers and test automation specialists
- Development leads planning test strategy
- DevOps engineers implementing CI/CD
- Quality assurance managers
- Team leads establishing testing standards

## Test Strategy Dimensions

- **Coverage Analysis**: Current coverage gaps, critical paths, edge cases
- **Test Pyramid**: Unit (70-80%), Integration (15-20%), E2E (5-10%)
- **Test Types**: Unit, Integration, E2E, Performance, Security, Smoke tests
- **Automation Framework**: Selection, organization, fixtures, reporting
- **CI/CD Integration**: Pipeline configuration, failure handling, regression detection

## Test Coverage Roadmap

Four-phase approach:
1. **Critical Path Testing** (Week 1-2): Essential workflows
2. **Unit Test Expansion** (Week 3-6): Business logic coverage
3. **Integration Tests** (Week 7-10): Component interactions
4. **Continuous Improvement** (Ongoing): Maintenance and enhancement

## Coverage Goals

- Unit Tests: >70% of application code
- Overall Test Suite: >80% code coverage
- Critical Path: 100% E2E coverage
- Performance: Baseline and regression detection
- Security: Vulnerability scanning included

## Deliverables

1. Test Strategy Document
2. Test Plan by Component
3. Implementation Roadmap
4. Test Case Catalog
5. CI/CD Integration Guide
6. Automation Best Practices Guide

## Use Cases

- Test strategy planning
- Coverage gap analysis
- Test automation architecture design
- CI/CD pipeline integration
- Regression prevention
- Team training on testing practices
- Quality metrics establishment
