# Systemic Risk Analysis Mode

## Overview

The Systemic Risk Analysis mode is specialized for quantitative analysis of systemic corruption, accountability cascades, and evidence-based risk modeling. It produces forensically rigorous reports suitable for policy audiences and accountability frameworks.

## Files

- **`accountability-analyst.agent.md`** - Primary analysis agent with full specification

## Target Audience

- Policy researchers and think tanks
- Accountability advocates and oversight bodies
- Risk analysts and financial auditors
- Government transparency initiatives
- International development organizations

## Key Capabilities

### Evidence Rigor
- Event-level evidence collection with specific citations
- Confidence tier assignment (High/Medium/Low)
- Primary source verification and gap flagging
- Unverified estimate marking [UNVERIFIED]

### Model Transparency
- Explicit assumption justification
- Sensitivity analysis for key parameters
- Cascade depth and compounding rate rationale
- [SENSITIVITY RISK] flagging for high-impact inputs

### Scenario Analysis
- Baseline scenario documentation
- Partial intervention modeling (50% event recoding)
- Delayed accountability scenarios (t+2 timing)
- Full accountability counterfactual comparison
- SRS, R0, and impunity factor tracking across scenarios

### Analytical Completeness
- Weakest evidence chain link identification
- Double-counting detection across events
- Jurisdiction gap analysis (federal vs state vs civil)
- Reversibility vs irreversibility assessment

### Communication
- Policy-actionable stakeholder summaries
- "What this is NOT" misuse preemption sections
- Heat tables with numeric ranges (not just bands)
- Clear non-adjudication framing throughout

## Research-First Workflow

When analyzing systemic risks:

1. **Research Phase**
   - Query Google or DuckDuckGo for systemic risk frameworks in your domain
   - Gather academic research and policy papers on accountability
   - Collect primary source evidence and official reports

2. **Evidence Collection Phase**
   - Verify all evidence with specific citations
   - Assign confidence tiers based on source quality
   - Flag unverified estimates and assumptions

3. **Analysis Phase**
   - Model cascade and accountability patterns
   - Perform sensitivity analysis on key parameters
   - Identify weakest evidence chain links

4. **Reporting Phase**
   - Generate policy-actionable summaries
   - Provide scenario analysis and counterfactuals
   - Clearly frame as analysis, not adjudication

## Output Standards

- Full markdown reports with changelog
- Citation format: [Source type | Institution | Date]
- [MODELED INFERENCE] flagging for all inferences beyond evidence
- No removal of existing appendices - only extensions
- Conflict markers for contradictory data
