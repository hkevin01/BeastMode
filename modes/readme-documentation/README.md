# README Documentation Mode

## Overview

The README Documentation mode specializes in creating, enhancing, and maintaining exceptional project README files that serve multiple audiences and provide clear guidance for users, contributors, and maintainers.

## Files

- **`readme-enhancer.agent.md`** - README documentation specialist

## Target Audience

- Project maintainers and leads
- Documentation specialists
- Open source contributors
- New project setups
- Onboarding coordinators

## README Scope

This mode covers:

- **First Impressions** - Clear project description and quick overview
- **Installation** - Multiple setup methods (npm, Docker, source)
- **Quick Start** - Get running in < 5 minutes
- **Usage & Examples** - Multiple real-world examples
- **Architecture** - System design and component explanation
- **API Reference** - Complete interface documentation
- **Configuration** - All settings and options
- **Development** - Setup for contributors
- **Testing** - Test coverage and running tests
- **Contributing** - How to submit contributions
- **License & Acknowledgments** - Legal and credits

## 16-Section Structure

1. Header & Visual (Project identity)
2. Table of Contents (Navigation)
3. Overview (Problem & solution)
4. Features (What it does)
5. Installation (Getting it)
6. Quick Start (First 5 minutes)
7. Usage & Examples (How to use)
8. Architecture (How it works)
9. API Reference (Detailed interface)
10. Configuration (Settings)
11. Development (For contributors)
12. Testing (Quality assurance)
13. Roadmap (Future direction)
14. Contributing (How to help)
15. License (Legal)
16. Acknowledgments (Credits)

## Quality Standards

### Completeness
- Answers: What? Why? How? Where?
- All sections customized for project type
- Working code examples (copy-paste ready)
- Multiple installation methods

### Clarity
- Audience-appropriate language
- Technical terms explained
- Clear examples with output
- Consistent formatting

### Functionality
- All links working
- All examples tested
- Code syntax highlighting
- Proper markdown formatting

### Navigation
- Table of contents with links
- Badges showing status
- Clear section hierarchy
- Links to related sections

## Audience Testing

Does it answer within each time frame?
- **10 seconds**: What does this project do?
- **1 minute**: Do I need it?
- **2 minutes**: How do I install it?
- **5 minutes**: How do I use it?
- **15 minutes**: How do I extend/contribute?
- **Always obvious**: Where do I get help?

## Use Cases

- Creating new project README from scratch
- Improving existing README documentation
- Adding missing sections or details
- Reorganizing for better structure
- Creating supporting docs (CONTRIBUTING, FAQ, API reference)
- Updating README for new features/versions
- Onboarding documentation overhaul
- Open source project quality improvement

## Project Type Customization

- **Libraries**: Focus on API, installation, quick start
- **Applications**: Focus on features, installation, usage
- **Frameworks**: Focus on architecture, examples, extensions
- **Tools/CLI**: Focus on commands, options, examples
- **Packages**: Focus on imports, usage patterns, examples
