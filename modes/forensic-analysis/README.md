# Forensic Analysis Mode

## Overview

The Forensic Analysis mode is specialized for detailed investigation, trace analysis, and forensic evidence evaluation. It combines deep codebase inspection with rigorous documentation of findings.

## Target Audience

- Security researchers and auditors
- Code forensics and reverse engineering specialists  
- Incident response teams
- Compliance and governance auditors
- Open-source security maintainers

## Key Capabilities

### Forensic Rigor
- Detailed trace analysis with execution paths
- Evidence collection with exact line numbers and file locations
- Impact assessment with quantified severity
- Chain-of-custody documentation

### Investigation Completeness
- Root cause analysis (not just symptoms)
- Affected component inventory
- Dependency and cascade impact mapping
- Timeline reconstruction

### Communication
- Forensic report generation with findings hierarchy
- Clear distinction between confirmed vs suspected issues
- Reproducible steps and test cases
- Remediation guidance with priority levels

## Typical Use Cases

- Code audit and vulnerability assessment
- Git history forensics and blame analysis
- Security incident reconstruction
- Compliance gap analysis
- Performance degradation investigation
- Dependency vulnerability mapping
- Data flow tracing for privacy audits

## Research-First Workflow

When conducting forensic analysis:

1. **Research Phase**
   - Query Google or DuckDuckGo for forensic methodologies and best practices
   - Gather incident response frameworks and checklists
   - Review compliance and evidence handling standards

2. **Evidence Collection Phase**
   - Trace execution paths and data flows
   - Document findings with exact file paths and line numbers
   - Establish chain of custody for evidence items

3. **Analysis Phase**
   - Perform root cause analysis
   - Map affected components and dependencies
   - Quantify severity and impact

4. **Reporting Phase**
   - Generate forensic reports with findings hierarchy
   - Include reproducible test cases
   - Provide prioritized remediation guidance

Forensic agents typically have access to:
- read, search, codebase (core analysis)
- web, fetch (reference gathering)
- usages, changes (impact analysis)
