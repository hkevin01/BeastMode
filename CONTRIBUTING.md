# Contributing

Thanks for improving BeastMode.

## Workflow

1. Fork the repository.
2. Create a focused branch.
3. Make small, reviewable commits.
4. Run local checks relevant to your change.
5. Open a pull request with a clear summary.

## Branch Naming

- feature/<short-description>
- fix/<short-description>
- docs/<short-description>

## Pull Request Checklist

- Change is scoped and documented.
- README is updated if behavior or structure changed.
- No secrets or credentials added.
- New files follow repository naming and style.

## Documentation Changes

For changes to agent profiles or instructions:

- Keep instructions actionable and testable.
- Avoid contradictory rules.
- Keep examples up to date with available tools.
