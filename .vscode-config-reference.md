# BeastMode VS Code Configuration

This file documents the exact VS Code configuration needed to use BeastMode agents across all your projects.

## Required Setting

Add this to **every** project's `.vscode/settings.json`:

```json
{
  "chat.agentFilesLocations": [
    "/home/kevin/Projects/BeastMode"
  ]
}
```

## Automated Setup

Instead of manual configuration, use the provided setup script:

```bash
cd /path/to/your/project
/home/kevin/Projects/BeastMode/setup-beastmode.sh
```

This script will:
1. Create `.vscode/settings.json` if it doesn't exist
2. Add/update the `chat.agentFilesLocations` setting
3. List all available BeastMode agents
4. Preserve any existing settings in `.vscode/settings.json`

## Manual Setup (Alternative)

If the script doesn't work for your environment:

1. Navigate to your project root
2. Create `.vscode/settings.json` if it doesn't exist
3. Ensure it contains the `chat.agentFilesLocations` setting (shown above)
4. Save the file
5. Reload VS Code (Cmd+Shift+P → "Developer: Reload Window")

## Recovery if Settings Are Lost

If updating VS Code removes the BeastMode configuration:

**Option 1: Re-run the setup script**
```bash
cd /path/to/your/project
/home/kevin/Projects/BeastMode/setup-beastmode.sh
```

**Option 2: Manual recovery**
1. Open `.vscode/settings.json`
2. Add the required setting (shown above)
3. Save and reload VS Code

## Verifying Setup

After configuration, verify BeastMode is properly set up:

1. Open a project folder in VS Code
2. Open Copilot Chat (Cmd+Shift+I / Ctrl+Shift+I)
3. Look for a custom agent dropdown or selector
4. You should see BeastMode agents listed

## Persistence Notes

- Store this file in your BeastMode repository for recovery purposes
- If VS Code settings keep getting reset, consider:
  - Using workspace-level settings (`.vscode/settings.json`) instead of user settings
  - Checking if settings are being auto-generated by a tool or script
  - Creating a `.vscode/extensions.json` to pin the Copilot extension version
